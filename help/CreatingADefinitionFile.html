Creating a definition file
    
The circuit definition file must be in a plain text format (e.g. `.txt'), and must conform to the grammar specification and the syntax rules. The easiest way to create a definition file is to make a copy of one of the example files, and modify it to meet your needs. The syntax is designed to be readable so shouldn't require much explanation, but the key components are:
  • DEVICES All the devices to be used in the simulation must be declared before they can be used in the other sections.
  • CONNECTIONS This defines the connections between the devices in the simulation. Each device input must be connected to exactly one output (from another device or itself).
  • MONITOR This defines the points which will be monitored and displayed as waveforms by the simulator. This can be easily modified by the GUI, so isn't vital to get correct at this stage.
